    Intuitive, clean, concise syntax

    Malina uses equality sign to assign value to element. No quotes needed to define a value.

    Just type your data and forget about closing tags and string brackets. It saves your time and also improves readability.

    Malina uses indents to specify blocks of nested elements like in popular programming language Python.

    Colon is required in the end of the parent element to prevent accidental typos in the document structure.


    Compatibility with XML and JSON

    Malina supports XML-specific entities like Attributes and Namespaces as well as JSON-specific entities like Arrays.

    In Malina, attributes are defined in the same way as elements with one difference, that the name of the attribute starts with @.

    All child attributes of the element have to be defined before the first child element. Code highlighting helps to distinguish between attributes and elements.

    Declare a namespace after #. Namespace can be declared for the whole file or just for the particular document. Put the namespace prefix before the element name separating them with dot.

    Namespaces are defined in the beginning of the file or document which make it easier to find namespace declaration in the file. Namespace prefix is separated with element by dot like in many programming languages.

    In Malina, an array is defined as a block having only array items or values. Name/value pairs are not allowed.

    All types of JSON values are supported including string, number, and literals true, false and null.

    Array item is an element with an empty name.

    This is an example of the array with 3 objects.


    Code reuse

    Malina has powerful code reuse features. You can create Alias for any fragment of your data and then use it in multiple documents. The alias definition starts with !$ . Reference alias in the document by typing $ followed by alias name.

    The name of the alias could be compound. The compound alias name can be used to create a hierarchy of aliases grouped by type.

    You will find aliases even more useful if you will use parameterized aliases. Put parameter anywhere in the alias definition by typing % followed by parameter name. You can also assign a default value.

    A parameter can represent either value of the element or element or even block of elements. The parameter representing a value called "value parameter".
    The parameter representing element or block of elements called block parameter.Both types of parameters can have a default value. You can define as many parameters as you want.
    Parameter with the same name can be defined several times. In this case parameter should have same type (value or block) in all places there it was defined.

    Reference parameterized alias in the document and pass real values in arguments defined in the alias body.

    You must specify the value for the argument if the corresponding parameter doesn't have the default value.

    Aliases can be nested inside each other so you can even create a template for a whole document.

    Using compound alias names you can group templates by categories so it will be easier to find them in future by name.

    Then you can use the template along with other already created aliases for lighting fast composing of your XML or JSON document.

    The created Malina document will be also significantly more readable comparing to the XML document it's representing.


    Cross-platform IDE

    We integrated Malina with MonoDevelop (developed by Xamarin, a subsidiary of Microsoft), an open source integrated development environment for Linux, OS X, and Windows.

    MonoDevelop is free, so there will be no extra costs for customers purchased Malina license.

    Organize your Malina files in solutions and projects.

    Create address related aliases in one file and product related aliases in another so it will be easier to find them in the future. Use aliases when creating a new or editing existing XML document. All defined aliases are visible for all files in the project.

    Malina project can also reference another Malina project so it is possible to share aliases between projects efficiently.

    Work like with mainstream programming language.

    Add schema files and references, navigate and manage your project files using Solution windows in MonoDevelop IDE.

    MonoDevelop is also integrated with established source control systems.

    Syntax highlighting and code folding improve readability of code.

    Fold blocks of code to navigate through the code faster.

    Autocomplete accelerates document creation and editing.

    Declare namespaces using autocomplete.

    Reference schema file in the project to see a list of available namespaces in the autocomplete list.

    Add elements and attributes using autocomplete.

    Reference schema file in the project to see the list of available elements and attributes in the autocomplete list.

    Add aliases using autocomplete.

    Aliases have to be declared in the same file, project or referenced project.

    Add arguments using autocomplete.

    You will see the list of available arguments in the autocomplete list if the cursor is inside of an alias.

    Interactive errors are displayed during document editing. Compilation errors are shown after compilation.

    An interactive error is displayed as a red underline. Put mouse cursor above the interactive error to see the error message.

    Generation of XML and JSON files

    Easy conversion from XML to Malina.

    We made it easy to convert XML to Malina and vice versa. Just copy XML to buffer and paste it in Malina format. It will be automatically converted.

    You can also convert the whole XML file to Malina and add it to the project. In this case use command "Add Malina file from XML".

    Easy conversion from Malina to XML.

    We made it easy to convert Malina to XML also. Just use "Copy Document as XML" to copy document to buffer and paste it in XML format in another text editor.

    If Malina solution is built successfully then all Malina documents are getting compiled into XML files. The XML files could be found in the project output folder.

    You can convert the whole document as well as just document fragment. To convert a fragment of the document to XML, select the fragment and use menu "Copy Selection as XML".



    Pattern Matching for Data Analysis

    When converting from XML or JSON to Malina, the conversion logic can automatically apply all known aliases and add them to the result document.

    It can be used to highlight certain data patterns in the document and make it easier to find or edit.
