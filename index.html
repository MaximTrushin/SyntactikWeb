Problem being solved

Verbosity

XML and JSON formats are verbose and have poor readability. Closing tags, brackets, and quotes slow down editing and make document harder to read.

No Code Reuse

Both formats are also missing even basic code reuse features which could improve both readability and editing of the documents.

Use Cases

Editing

Create data documents using human-friendly syntax. Generate XML or JSON file based on Malina document.

Reading

Automatically convert XML or JSON data into more readable Malina format and explore your data efficiently.

Code reuse

Create reusable templates and snippets of code and reuse them in your future documents.

Code sharing

Share your code templates with a team and use templates and snippets created by team members.

Data consolidation

Copy XML or JSON data in the buffer and paste it in Malina format. Consolidate data from different sources in one format.

Data analysis

Find data matching the predefined templates in the document.
